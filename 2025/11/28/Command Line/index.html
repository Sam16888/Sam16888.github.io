Mac命令列(Command Line)基本指令

發表於2025-11-28|更新於2025-12-10
|瀏覽量:

操控電腦的介面與方式：

  • GUI(Graphical User Interface)圖形使用者介面 ： 使用滑鼠點按或手指觸碰圖像與作業系統互動的介面，如Windows桌面、macOS桌面和智慧型手機桌面等等。
  • CLI(Command-Line Interface)命令列介面 ： 使用鍵盤輸入指令與作業系統互動的介面，也有人稱之為CUI(Character User Interface)文字使用者介面，如Windows的命令提示字元、Linux/macOS的終端機。

重要目錄(For macOS)

  • 根目錄(/)：macOS的第一個資料夾，包含系統相關的資料夾。
  • 使用者目錄(/Users)：包含所有使用者的家目錄。
  • 家目錄(~ 等同於 /Users/你的使用者名稱 )：每個使用者存放個人檔案的資料夾。打開終端機時，預設位置為家目錄。

絕對路徑與相對路徑

  • 絕對路徑：從檔案系統的根目錄開始的完整路徑。在Unix/Linux中是「 / 」開頭，在Windows中是「 C:\ 」開頭。
    如：/Users/你的使用者名稱/Desktop
  • 相對路徑：從當下工作目錄開始，指定一個檔案或目錄相對於目前位置的路徑，該檔案或是目錄必須包含於工作目錄。
    如：假設終端機當下工作目錄是家目錄( ~ 或 /Users/你的使用者名稱 )，那麼 Desktop 即為表示當下工作目錄下的 Desktop 資料夾。
  • 如何選擇？
    當你需要一個能指向特定位置的、最確定的方式時(例如從網頁中連結到一個在特定網路位置的檔案(如 http://…)，或者在某些系統設定或程式中使用完整路徑)，可以使用絕對路徑。
    當你建立一個包含多個檔案(例如網頁、CSS、圖片)的專案，並希望整個專案可以被移動而不用修改每個連結時，可以使用相對路徑(更為方便且靈活)。
  • 目錄與路徑的參考資料如下：
    【Flutter 學習筆記】macOS 終端機 zsh 基本指令操作教學

基本指令

  • pwd(告訴你我在哪裡)
    以下指令介紹都是先假設終端機的位置是在某一個工作目錄之下，不在特別說明(都是寫「在提示符後面…」)。

    • 全名為Print Working Directory，用於顯示使用者目前所在目錄(資料夾)位置的絕對路徑。在你執行其他指令(如 ls 或 cd)之前，或是在你不確定當前目錄時，即可使用該指令確認你所在的位置。
    • 在提示符後面輸入pwd並按Enter鍵執行，終端機就會顯示該工作目錄的絕對路徑。

  • ls(show出檔案的清單)

    • 全名為list，用於列出使用者目前所在目錄的所有檔案和資料夾。
    • 在提示符後面輸入ls並執行，終端機就會顯示該工作目錄的所有檔案和資料夾。
    • 若要顯示更詳細的資訊，可以搭配其他參數，如下所示。
      • ls -a：顯示包含隱藏檔案（以 . 開頭的檔案）的列表
      • ls -l：以長格式顯示詳細資訊，包括權限、擁有者、檔案大小和修改時間
      • ls -al(或ls -la)：結合以上兩個參數，顯示所有檔案（包含隱藏檔）的詳細列表
      • ls -t：依照最後修改時間的先後順序來排序檔案

  • cd(帶你飛到各處)

    • 全名為Change Directory(切換目錄)，用於在不同目錄(資料夾)之間移動。
    • 如果你想往內層的目錄移動，那麼在提示符後面輸入cd 內層目錄之相對或絕對路徑並執行，終端機就會切換至該工作目錄之內的目錄。
    • 如果你想往外層的目錄移動，那麼在提示符後面輸入cd 外層目錄之絕對路徑並執行，終端機就會切換至該工作目錄之外的目錄。
    • 常用的cd指令 :
      • cd / : 切換到根目錄
      • cd ~ : 切換到家目錄
      • cd … : 切換到上一層(外面一層)目錄

  • man(帶你認識指令)

    • 全名為Manual(使用手冊)，用於查詢指令的使用說明(詳細資訊與進階用法)，僅限於Unix和Linux的終端機。
    • 在提示符後面輸入man 指令名稱並執行，終端機就會顯示該指令的詳細用法和參數說明，按下 q 鍵即可退出說明頁面。比如執行man ls即可查閱上述四種ls參數的資訊。

  • clear(一鍵清空！)

    • 用於清空終端機的畫面
    • 在提示符後面輸入clear並執行，終端機的畫面就會乾淨溜溜。
    • 相同效果的兩種快捷鍵：Command + K 或 Control + L

檔案操作相關指令

  • touch

    • 用於建立新的空白檔案(較常使用)或更新現有檔案的存取及修改時間。
    • 在提示符後面輸入touch 不存在的檔案名稱(新的檔案名稱).副檔名並執行，終端機就會在該工作目錄之下建立一個新的檔案。
    • 在提示符後面輸入touch 已存在的檔案名稱(舊的檔案名稱).副檔名並執行，終端機就會更新該檔案的存取時間(atime)和修改時間(mtime)為當前時間。
      可以理解成我「碰(touch)」了一下這個檔案，讓電腦認為它被我修改過。
    • 建立多個不同副檔名的檔案：
      touch 第1個檔案名稱1.副檔名1 第2個檔案名稱.副檔名2 ... 第n個檔案名稱.副檔名n(n為正整數)
      例如：touch index.html style.css script.js，執行該指令後，當前目錄會建立前述3個檔案。
    • 建立多個相同副檔名的檔案：
      • 序列形式
        1. 數字序列：touch 檔案名稱{1..n}.副檔名，
          建立 n 個檔案，n為大於1的正整數。
          例如：touch text{1..5}.txt，執行該指令後，當前目錄會建立5個文字檔，如下所示。
          text1.txt, text2.txt, text3.txt, text4.txt, text5.txt
        2. 英文字母序列：touch 檔案名稱{a..第n個小寫英文字母}.副檔名(大小寫皆可)，
          建立 n 個檔案，n為2~26的任意正整數。
          例如：touch text{A..E}.txt，執行該指令後，當前目錄會建立5個文字檔，如下所示。
          textA.txt, textB.txt, textC.txt, textD.txt, textE.txt
      • 連乘形式：touch 檔案名稱_{1..M1}_{1..M2} ... _{1..Mn}.副檔名，
        建立 Ｍ1 X M2 X … X Mn 個檔案(建立巨量檔案)，M1、Ｍ2…Mn皆為大於1的正整數。
        上述是以全為數字序列的形式說明，你也可以全為英文字母序列的形式或是兩種序列交錯使用。
        例如：touch text_{a..c}_{1..3}.txt，執行該指令後，當前目錄會建立3X3=9個文字檔，如下所示。
        text_a_1.txt, text_a_2.txt, text_a_3.txt,
        text_b_1.txt, text_b_2.txt, text_b_3.txt,
        text_c_1.txt, text_c_2.txt, text_c_3.txt

  • rm

    • 全名為Remove，用於刪除檔案或資料夾(目錄)。
    • 在提示符後面輸入rm 檔案名稱.副檔名並執行，終端機就會在該工作目錄之下刪除該檔案。
    • 在提示符後面輸入rm -r 資料夾名稱並執行，終端機就會在該工作目錄之下刪除該資料夾及其所有內容。
    • 刪除多個檔案或資料夾
      • rm 第1個檔案名稱.副檔名1 第2個檔案名稱.副檔名2 ... 第n個檔案名稱n.副檔名n：一次刪除當前目錄之下n個檔案，n為大於1的正整數。
      • rm -r 第1個資料夾名稱 第2個資料夾名稱 ... 第n個資料夾名稱：一次刪除當前目錄之下n個資料夾及其所有內容，n為大於1的正整數。
    • 其他常用的rm指令
      • rm *.副檔名：一次刪除當前目錄之下所有該副檔名的檔案，例如 rm *.html 會刪除當前目錄之下所有副檔名為html的檔案。
      • rm -i 檔案或資料夾名稱：在刪除檔案或資料夾之前，會提醒使用者確認是否刪除。
      • rm -f 檔案或資料夾名稱：直接強制刪除檔案或資料夾，不會先提醒使用者。
        這邊說明一下，因為使用rm指令刪除的東西會徹底消失(不會跑去垃圾桶)，所以建議養成使用rm -i 刪除檔案及資料夾。

  • mkdir

    • 全名為Make Directory，用於建立資料夾(目錄)。
    • 在提示符後面輸入mkdir 資料夾名稱並執行，終端機就會在該工作目錄之下建立一個該名稱的新資料夾。
    • 建立多個資料夾
      • 數字序列：mkdir 資料夾名稱{1..n}，建立n個新資料夾，n為大於1的正整數。
        例如：mkdir folder{1..5}，執行該指令後，當前目錄會建立5個新資料夾，如下所示。
        folder1, folder2, folder3, folder4, folder5
      • 字母序列：mkdir 資料夾名稱{a..第n個小寫英文字母}(大小寫皆可)，建立n個新資料夾，n為2~26的任意正整數。
        例如：mkdir folder{A..E}，執行該指令後，當前目錄會建立5個新資料夾，如下所示。
        folderA, folderB, folderC, folderD, folderE
    • 如果你想刪除「 空資料夾 」，可以使用rmdir 空資料夾名稱。

  • mv

    • 全名為Move，用於移動檔案或資料夾，亦可變更檔案名稱(重新命名檔案)。

    • 如果你在提示符後面輸入mv 來源檔案名稱.副檔名或資料夾名稱(/) 目的地資料夾名稱(/)並執行，終端機就會在該工作目錄之下將來源檔案或資料夾移至目的地資料夾。

    • 如果你在提示符後面輸入mv 來源檔案名稱.副檔名或資料夾名稱(/) ..(/)並執行，終端機就會在該工作目錄之下將來源檔案或資料夾移至上一層資料夾。

    • 如果你在提示符後面輸入mv 舊檔案名稱.副檔名 新檔案名稱.副檔名並執行，終端機就會在該工作目錄之下將舊檔案名稱變更為新檔案名稱。

    • 如果你在提示符後面輸入mv 舊資料夾名稱(/) 新資料夾名稱(/)並執行，終端機就會在該工作目錄之下將舊資料夾名稱變更為新資料夾名稱。
      PS：(/)表示/可有可無

    • 如果你在提示符後面輸入mv 舊檔案名稱.副檔名 ../新檔案名稱.副檔名並執行，終端機就會在該工作目錄之下將舊檔案名稱變更為新檔案名稱並移至上一層資料夾。
      這便特別說明一下，沒打「/」的話(mv 舊檔案名稱.副檔名 ..新檔案名稱.副檔名)，舊檔案會徹底消失喔！

  • cp

    • 全名為Copy，用於複製檔案或資料夾。
    • 如果你在提示符後面輸入cp 來源檔案名稱.副檔名 新檔案名稱.副檔名並執行，終端機就會在該工作目錄之下將來源檔案複製一份並命名為新檔案名稱；
    • 如果你在提示符後面輸入cp 來源檔案名稱.副檔名 目的地資料夾名稱(/)並執行，終端機就會在該工作目錄之下將來源檔案複製一份至目的地資料夾；
    • 如果你在提示符後面輸入cp -r 來源資料夾名稱 目的地資料夾名稱(/)並執行，終端機就會在該工作目錄之下將「來源資料夾」複製一份至目的地資料夾；
    • 如果你在提示符後面輸入cp -r 來源資料夾名稱/ 目的地資料夾名稱(/)並執行，終端機就會在該工作目錄之下將「來源資料夾的內容」複製一份至目的地資料夾；
      PS：(/)表示/可有可無

Vim

  • Vim為macOS終端機的內建文字編輯器。
  • 如果Vim搭配一些插件就可以當成另一種VS Code或Sublime Text，可以在Vim裡面寫程式。如果你正在操作一台沒有文字編輯軟體的電腦，那麼你就可以用終端機的Vim進行文字編輯。
  • 以下只說明常見的操作，想了解更多關於Vim的介紹可以參考我提供的網站或是自行谷歌。
    • 【Vim 編輯器 入門指南 (上)】用思維的速度寫程式
    • 【Vim 編輯器 入門指南 (下)】用程式來寫程式
  • 基本操作(假設終端機的位置是在某一個工作目錄之下)：
    • 進入 Vim：在終端機輸入 vim 該目錄之下的某一個檔名.副檔名，若該檔案不存在，則會建立新檔案。
    • 退出編輯模式：按 Esc 鍵
    • 進入插入模式：預設為命令模式，按 i 或 a (在游標後方插入) 可進入插入模式，可開始輸入文字。
    • 儲存與退出(命令模式)：
      • ：w：儲存檔案
      • ：q：退出 Vim (若有未儲存的修改，需使用 :q! 強制退出。)
      • ：wq 或：x：儲存並退出
      • ：q!：不儲存，強制退出。
  • 進階操作：
    • 游標移動：
      • h：向左
      • j：向下
      • k：向上
      • l：向右
    • 編輯操作：
      • dd：刪除當前行
      • yy：複製當前行
      • p：在游標下方貼上複製的內容
      • u：復原上一個動作
    • 搜尋: /搜尋文字：向下搜尋。

其他好用指令

  • cat

    • 有四個功能：顯示檔案內容、顯示帶行號的內容、建立新檔案和合併檔案。

    • 顯示檔案內容：在提示符後面輸入cat 檔案名稱.副檔名並執行，終端機就會一次性顯示整個檔案內容。

    • 顯示帶行號的內容：在提示符後面輸入cat -n 檔案名稱.副檔名並執行，終端機就會一次性顯示整個檔案內容並帶行號。
      例如：cat -n Hello.py
      1 name = input("請輸入你的名字 ： “)
      2 print(f"你好，{name}”)

    • 建立新檔案：在提示符後面輸入cat > 新建檔案名稱.副檔名並執行，終端機就會在該工作目錄之下建立一個新的檔案。此時你可以在指令碼的下方敲鍵盤輸入文字，直到按下 Ctrl + D 結束，輸入的內容會寫入新檔案中。

    • 合併檔案：在提示符後面輸入cat 第1個檔案名稱.副檔名 第2個檔案名稱.副檔名 ... 第n個檔案名稱n.副檔名 > 容器檔案名稱.副檔名(內容不同但副檔名相同的n個檔案)並執行，終端機就會在該工作目錄之下將n個檔案的內容合併到一個容器檔案中，n為大於1的正整數。
      例如：cat chapter1.txt chapter2.txt > book.txt，將chapter1.txt和chapter2.txt內容合併寫入book.txt。

  • grep

    • 全名為Global Regular Expression Print(中譯為全文正則表達式列印)，是一種文本搜尋工具，用於在檔案中搜尋文字字串。
    • 如果你在提示符後面輸入grep 關鍵字詞 檔案名稱.副檔名並執行，終端機就會一次性顯示整個檔案內容的所有關鍵字詞。
    • 如果你在提示符後面輸入grep -i "英文關鍵字詞" 檔案名稱.副檔名並執行，終端機就會一次性顯示整個檔案內容的所有大小寫的英文關鍵字詞(忽略大小寫搜尋)。
    • 進階用法：
    • 於整個資料夾遞迴搜尋一個關鍵字詞：grep -r "keyword" ./：從當前目錄(.)開始，遞迴搜尋包含 keyword 的所有檔案及其關鍵字詞。
    • 於整個資料夾遞迴搜尋多個關鍵字詞：grep -r -e "keyword1" -e "keyword2"... -e "keywordn" ./：從當前目錄(.)開始，遞迴搜尋包含 keyword1 或 keyword2 … 或 keywordn (至少有其中一個關鍵字詞即可)的所有檔案及其關鍵字詞，n為大於2的正整數。
    • 如果上述「-r」改成「-rl」，遞迴搜尋的結果只會顯示檔案名稱.副檔名；
      如果上述「-r」改成「-irl」，不分英文字母大小寫進行遞迴搜尋，結果只會顯示檔案名稱.副檔名。

  • wget

    • 全名為World Wide Web Get，用於抓取(下載)網址的資源，可以是網頁、影片、圖片、音訊檔案或文件。
    • 因為macOS終端機沒有預設該指令，所以需要另外安裝。建議使用Homebrew安裝，在終端機下輸入以下指令brew install wget。
    • 如果你在提示符後面輸入wget 資源網址並執行，終端機就會幫你下載該資源存放於該工作目錄。
    • 如果你想查閱下載檔案，在提示符後面輸入open 下載檔案名稱.副檔名，終端機就會使用適當的應用程式開啟下載檔案的內容。

  • echo

    • 用於輸出文字、字串或變數的值到螢幕上
    • 如果你在提示符後面輸入echo "字串內容(亦可空白)"並執行，終端機就會顯示兩個雙引號內的字串內容。
    • 這邊只簡述輸出文字，其他用法請自行研究。

指令膠水(指令的組合技)

  • redirection(> 或 >>)

    • 中譯為重新導向，用來將前述指令的結果寫入於後述的文字檔之中並儲存。
    • 「>」表示覆蓋寫入：
      如果後述的文字檔是新建檔案，則前述指令的結果會被寫入該文字檔；如果後述的文字檔是既有(已存在)的檔案，則前述指令的結果會被「覆蓋寫入」於該文字檔，原先文字檔內容會變成前述指令的結果。
    • 「>>」表示附加寫入：
      如果後述的文字檔是新建檔案，則前述指令的結果會被寫入該文字檔；如果後述的文字檔是既有的檔案，則前述指令的結果會被「附加寫入」於該文字檔，原先文字檔內容的後方會新增前述指令的結果。
    • 以下直接以實例說明，假設終端機的位置是在某一個工作目錄之下，
      如果你在提示符後面輸入ls -al > list_result.txt(新建文字檔)並執行，終端機就會將該目錄的檔案資訊寫入到list_result.txt，並存放於該目錄之中。
    • 如果你在提示符後面輸入echo “123” > 123.txt(新建文字檔)並執行，終端機就會將雙引號的內容寫入於123.txt，並存放於該目錄之中。
    • 承上，如果你在提示符後面輸入echo “456” > 123.txt(既有文字檔)並執行，終端機就會將雙引號的內容覆蓋寫入於123.txt，並存放於該目錄之中。123.txt內容為456(456覆蓋123)。
    • 承上，如果你在提示符後面輸入echo “789” >> 123.txt(既有文字檔)並執行，終端機就會將雙引號的內容附加寫入於123.txt，並存放於該目錄之中。123.txt內容為為456隔行789(隔行新增789)。

  • pipe( | )

    • 中譯為管道，將前一個指令的標準輸出作為後一個指令的標準輸入，形成一個數據處理鏈(指令鏈)，實現「串流」處理，讓多個指令協同工作。
    • 以下直接以實例說明，假設終端機的位置是在某一個工作目錄之下，
      如果你在提示符後面輸入history | grep "A"並執行，終端機就會查看歷史指令，並篩選出包含A的特定指令，然後顯示於介面上。
    • 如果你在提示符後面輸入ls -l | grep "A"並執行，終端機就會先查看該目錄下的所有檔案資訊，並使Ａ這個字母上色，然後顯示於介面上。
    • 如果你在提示符後面輸入cat iamtext.txt | grep "C"並執行，終端機就會直接從iamtext.txt抓出關鍵字Ｃ並顯示於介面上。
    • 如果你在提示符後面輸入cat iamtext.txt | grep "C" > result.txt並執行，終端機就會直接從iamtext.txt抓出關鍵字Ｃ並寫入result.txt。

心得：

以上我是看完鋰學院CMD101的教學影片之後，又自行研究部分指令的其他功能，將實用的功能記錄下來。雖然沒有將每個指令介紹得非常詳細，但以上內容對於初學者來說非常夠用。如有錯誤，請不吝指教。
以下是思維轉變的小小插曲
由於現在的主流程式編輯器已具備許多提升開發效率的插件和AI工具，所以現今開發人員的敲鍵盤時間相較AI工具出現以前減少了許多，因此盲打不再是提升開發效率的關鍵因素。
這一句因果關係是我原本的想法，「現在不會盲打的我」也覺得寫程式一段時間之後，盲打能力一定會自然地提升，沒有必要花額外的時間刻意練習盲打，讓自己盡早上手。
當我的文章寫到Vim的時候，
因為看到參考連結的貼文裡寫到
「發揮編輯器Vim(其他IDE也是)的威力有一個必要前提，就是具備盲打能力。」，
然後我又想到「如果我學會了盲打，每天就能擠出更多的學習時間，轉職後也能擠出一些時間當薪水小偷，何樂而不為？」，
所以我決定從現在開始，每天學python之前先花一點時間練習盲打直至順手。
看來我終於有一個正當的理由去買一套機械式鍵盤了哈哈
