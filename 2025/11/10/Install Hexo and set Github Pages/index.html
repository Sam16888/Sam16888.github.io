使用Hexo和Github Pages架設個人部落格(主題為Butterfly)

發表於2025-11-10|更新於2025-11-15|安裝教學
|瀏覽量:

前言

我剛上大學的時候創建了一個Ｗordpress的部落格，用來紀錄在人生不同階段裡值得回憶的大小事。
現階段準備轉職軟工的我想找一個專門紀錄自己學習心得的部落格，
谷歌了一下就決定使用 Hexo + Github Pages 來建構學習網誌。

安裝Git(用Homebrew安裝)

打開iTerm2，執行brew install git，
接者執行git --version，顯示當下版本序號，
然後前往Git的官方網站查閱當下的序號是否為最新版本序號。
https://git-scm.com/downloads/mac
如果當下的序號不是最新版本，那麼在iTerm2接續執行以下三個步驟。

  1. 執行brew update，更新Homebrew，確保你使用的是最新版本的Homebrew和套件資訊。
    如果你有定期更新Homebrew的習慣，可以省略此步驟。
  2. 執行brew upgrade git，更新Git。
  3. 執行git --version，檢查當下版本(順利的話一定是最新版本序號)。
    PS:如果你沒有裝Homebrew，可以參考下方影片(這是我另外找的)，詳細解說安裝Homebrew的過程。
    在macOS安裝Homebrew，測試brew套件管理工具安裝軟體
    其中有一段安裝過程會跑比較久，請耐心等候，不是你的原生終端機當掉喔XD

安裝nvm(不建議用Homebrew安裝)

前網nvm的官方github，取得最新版本的nvm腳本。
https://github.com/nvm-sh/nvm
當下最新版本的nvm腳本
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
將其複製並貼在iTerm2上，按Enter執行安裝。
此外，因為官方提供的腳本被使用者執行時，它會自行判斷是安裝或是更新程序，所以往後更新方式就是上面提及的安裝方式。
PS:不建議使用Homebrew安裝nvm的原因
https://blog.marsen.me/2023/03/22/2023/dont_use_brew_install_nvm/

安裝Node.js

  1. 執行nvm ls-remote，列出可用的Node.js 所有版本。
  2. 執行nvm install "版本序號(v後面的字串)"(當下最新的LTS: Jod版本是22.19.0)，安裝你想要的Node.js版本。
    建議轉職軟工的初學者使用”當下最新的LTS版本”(可以去Node.js的官方網站查看)，為何要用這個版本？
    因為該版本提供了長期的安全性更新與錯誤修復，高穩定性的開發環境是初學者的必備利器。
  3. 執行nvm use "版本序號"，使用已安裝的Node.js 版本。


安裝Visual Studio Code

VS Code是之後撰寫文章或程式碼的最佳拍檔，建議可以先安裝。
後續也可以用它來編輯yml檔案。

註冊Github帳號

如果你有註冊過，可以省略此步驟。

安裝Hexo

  1. 執行npm install hexo-cli -g，透過 npm 安裝 Hexo。
  2. 執行hexo v(或是hexo version)，確認是否完成安裝。
  3. 執行hexo init "自取名稱"(自取放置hexo檔案的資料夾名稱)，初始化安裝 hexo。
    下文皆以“XXX”表示自取名稱為XXX的資料夾，可稱其為根目錄。

  1. 執行cd XXX，移動至根目錄。
  2. 執行npm install，安裝所需套件。
  3. 執行hexo s(或是hexo server)，建立第一個網站(預設主題為landscape)，點擊下方網址(http://localhost:4000/)，就會在預設瀏覽器顯示該頁面(首次打開hexo網站)。


將預設主題(landscape)更改為Butterfly

  1. 前往Butterfly的官方部落格，取得最新版本的Butterfly的腳本，建議選擇「Git安裝」。
    https://butterfly.js.org/posts/21cfbf15/
    git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly
    將其複製並貼在iTerm2上，按Enter執行安裝。根目錄的“themes”資料夾裡面會出現一個“butterfly”資料夾。

  1. 暫離iTerm2，打開檔案總管前往根目錄資料夾，接著用文字編輯器打開「_config.yml」檔案，
    找到內文下方的「theme: landscape」，將landscape改為butterfly並儲存該檔案。
    PS:也可以使用VS Code，打開yml檔進行編輯，程式碼識別度較高。

  1. 回到iTerm2，執行npm install hexo-renderer-pug hexo-renderer-stylus --save，
    安裝pug and stylus renderers(渲染器插件)。
  2. 執行cd XXX，移動至根目錄。
    若你當下的位置是在該資料夾，可以省略此步驟。
  3. 執行hexo s，你會看到用好幾個#寫的BUTTERFLY，表示更改主題成功。
    一樣點擊下方網址(http://localhost:4000/)，就會在預設瀏覽器顯示蝴蝶主題的頁面。


  1. 進入蝴蝶主題資料夾，複製「_config.yml」檔案，
    再將其複製檔的檔名改為「_config.butterfly.yml」，最後將其移至hexo的根目錄(就是"XXX"資料夾內)。
    為何要這麼作？因為可以減少之後更改主題設定帶來的不便(詳細內容見上方的官方部落格內文)。

設定Github SSH連線

  1. 在iTerm2執行下方兩行指令，
    git config --global user.name(你的Github用戶名稱)
    git config --global user.email(你的Github電子郵件)，
    配置你的Github用戶名稱和電子郵件。

  2. 接續執行下方兩行指令，
    git config user.name
    git config user.email，
    確認下方出現你的Github用戶名稱和電子郵件是否與上一步輸入的內容相同。

  3. 執行ssh-keygen -t rsa -C user.email，

    重點來了!重點來了!!重點來了!!!

    後續會出現3次Enter開頭的句子，每出現1次就按1次Enter，
    最後會跑出一行公共密鑰和隨機生成的公鑰辨識圖像。

  1. 在你的使用者資料夾內的帳號名稱資料夾內會出現".ssh"的資料夾，
    打開"id_rsa.pub"公鑰文字檔，複製裡面全部的內容。("id_rsa"文字檔是你的私人密鑰)


  1. 打開網頁瀏覽器至你的Github個人頁面，
    點選右上角個人頭像，點選列表中的Settings，
    點選SSH and GPG keys，點選New SSH key。
    Title欄位可以自訂名稱(我是寫hexo)，
    Key type選擇Authentication Key，
    Key欄位貼上剛才複製的內容，最後按Add SSH key。

  1. 執行ssh -T git@github.com（這不是user.email喔XD直接複製貼上即可），
    若出現「 Hi “你的Github用戶名”! You’ve successfully authenticated, but GitHub does not provide shell access. 」，表示連線成功。
    詳細內容可以參考下方網站
    https://hackmd.io/@yuna9068/B1vw4F9ta

將Hexo部屬至Github Pages

  1. 在你的Github頁面新增一個Repository，
    然後在你的Repository name輸入你的Github Pages子網域(你的Github帳號.github.io)。
    PS:Github Pages 會提供一組「你的Github帳號.github.io」的子網域，一個帳號僅有一組子網域。
  2. 前網Hexo根目錄，打開_config.yml，至最最底下更改Deployment的設定，如下圖所示。

PS:message 不是必要的設定項目。建議可以加這一項，因為它可讓你知道每一次部署的目的是什麼。
3. 打開iTerm2執行cd XXX，前往hexo的根目錄。
4. 執行hexo c(或是hexo clean)，清除暫存檔。
5. 執行npm install hexo-deployer-git --save，安裝deployer-git(hexo佈署套件)。
PS:若有出現可以更新npm的訊息，直接複製指令碼更新npm。
6. 執行hexo g -d(或是hexo generate -deploy)，生成Hexo靜態網頁，
並將其部署至Github Pages。
若出現「INFO Deploy done: git」，表示你的Hexo Blog成功部屬至Github Pages。

利與弊

為何我想用Hexo + GitHub Pages作為我的學習網誌？
就是因為他們有太多的優點，以及可以無視的缺點XD

Hexo 的優點 ：

  1. 專注寫作：支援 Markdown 語法，讓使用者專注於內容創作，而非排版和程式碼。
  2. 豐富的資源：提供上百種免費且現成的主題模板，也支援多樣的外掛，可以快速建立好看的部落格。
  3. 上手容易：有豐富的中文文件和教學資源，適合中文使用者。

GitHub Pages 的優點：

  1. 免費主機：提供免費的靜態網頁空間，不用額外支付主機費用。
  2. 穩定的託管服務: 是一個穩定且廣泛使用的網頁託管服務。
  3. 免費網域：預設會提供一個免費的 username.github.io 子網域。
  4. 版本控制：使用 Git 部署，可以輕鬆進行版本控制。
  5. 部署方便：可以直接將本地寫好的文章部署到 GitHub 上，省去繁瑣的伺服器環境建置和維護工作。

天底下沒有無料，也沒有任何限制條件的東西(除了A片之外XD)
享用免費、部署方便、安全、高效能和穩定性的Hexo + GitHub Pages之前，
一定要知道它的限制條件(也就是缺點)，如下所示 :

  1. 網站類型僅限靜態網頁：不支援需要伺服器端程式（如 PHP）或資料庫的動態網站
  2. 檔案大小上限為1GB：發布的網站檔案總計不得超過 1 GB
  3. 流量為每月100GB：這是流量的軟體限制，超出後可能會受到影響
  4. 每小時最多部署10次：每小時將Hexo網站推送到GitHub的次數不得超過10
  5. 必須使用公開儲存庫：GitHub Pages的源儲存庫必須是公開的
  6. 不可商業使用：免費的GitHub Pages不可作商業用途

基本上可以無視這些限制條件，它們不會造成很大的阻礙，分析如下：

  1. 對於網站類型而言，因為記錄學習心得的網誌內容本身就是靜態網頁，所以該限制條件可以無視。
  2. 對於檔案大小與流量而言，大部分個人部落格或作品集網站的使用量通常都低於這些限制條件。
  3. 對於部署頻率而言，在你完成文章寫作之後，再進行部署。你的部署頻率一定會遠低於限制條件。
  4. 對於儲存庫而言，由於網誌內容屬於知識與技術的交流(記得遵守版權的相關法律規範)，因此該限制條件也可以無視。
  5. 對於商業使用而言，基於記錄學習心得這件事與商業用途的關聯性不大，於是該限制條件當然可以無視。

心得

我花費至少15個小時才建立好個人網誌Hexo+Github Pages(與安裝個性化終端幾iTerm2差不多)，其實更麻煩的是後續butterfly的版面設計XD(花費時間取決於你想要的部落格環境)
我原先是谷歌搜尋前輩的一些相關心得文章，並按照他們的方式進行樣式修改，但後來才發現這樣做只會將原生的Butterfly部落格愈改愈亂，而且會衍生出一堆問題(部署失敗、Butterfly首頁僅呈現HTML檔、Butterfly分頁出現404頁面等等)。
於是我又刪掉原有的Hexo，再重新安裝一次，而這一次我就用最保險的方式：
前往Butterfly官方部落格進行樣式修改。
花費將近20個小時才將部落格打造成滿意的樣子，也就是你點進這篇文章之前的部落格首頁。
此外，網路上也有很多「魔改」Butterfly 的心得文章，可以使頁面呈現更酷炫的視覺與互動效果(例如：部落格頁面右下角有一隻可愛貓咪，貓咪的眼睛視線會隨著你的鼠標移動。)。
也因為有些魔改技巧會用到基礎前端3劍客(HTML、CSS、Javascript)的知識，而且我也很滿意目前改完樣式的Butterfly，所以我不會去魔改。如果你對魔改有興趣，可以自行爬文研究。
